# Vibe Coding Playbook

Real guides for building real things with AI. Not theory. Not "just tell your agent to..." Not another meta-guide about meta-guides.

This is how we actually build at [Mad House](https://github.com/madebymadhouse).

---

## What's here

| Section | What you'll find |
|---------|-----------------|
| [tools/](tools/) | Honest tool comparisons. Claude Code vs Cursor vs Windsurf vs Copilot vs Gemini CLI. |
| [setup/](setup/) | How to configure a project for AI from day one. Context files, gitignore, environment. |
| [workflow/](workflow/) | Starting right, keeping context, reviewing AI code, when to stop. |
| [case-studies/](case-studies/) | Real builds: chopsticks (Discord bot) and kinetrak (real-time body tracking). Actual prompts, actual failures. |
| [prompts/](prompts/) | Copy-paste prompts for setup, debugging, review, architecture, docs. |
| [scripts/](scripts/) | `create-vibe-project.sh` and `project-health.sh` -- tools you can use today. |

---

## Who this is for

- You're building something real and want to move faster with AI
- You've used Claude Code or Cursor but feel like you're leaving performance on the table
- You want honest trade-offs, not hype

## Who this is not for

- People who want a framework for thinking about AI
- People who want to read about vibe coding without vibe coding

---

## Start here

1. Read [tools/overview.md](tools/overview.md) to pick (or confirm) your tool
2. Read [setup/context-files.md](setup/context-files.md) before your next project
3. Grab [scripts/create-vibe-project.sh](scripts/create-vibe-project.sh) and run it

If you're mid-project and something is broken: [workflow/reviewing.md](workflow/reviewing.md) and [prompts/debugging.md](prompts/debugging.md).
